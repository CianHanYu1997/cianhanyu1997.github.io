Python的語言特性

介紹 Python 的幾個核心語言特性，包含動態強類型、is 與 == 的差異、以及可變與不可變型別的區別，
並了解這些設計背後的原因。

本文概要：

  • 動態強類型語言是什麼？
  • is 和 == 的差別
  • Dict 與 Tuple 的設計差異與使用時機
  • 可變 vs. 不可變型別一覽

1. 動態強類型語言

Python 是動態且強型別的語言：

  • 動態：不需要事先宣告變數型別，Python 會在執行時自動判斷。
  • 強型別：不會發生隱式型別轉換（例如 "1" + 1 在 Python 中會直接報錯）。

2. is 和 == 的差別

運算子 比較的是
== 值是否相等
is 是否指向同一個記憶體位址

小整數池（Integer Interning）：Python 預先建立 -5 ~ 256 的整數物件，
這個範圍內的整數會共用同一個記憶體位址，因此 is 比較結果為 True。
超出這個範圍後，每次賦值會建立新物件，is 就不再可靠了。

3. Dict 與 Tuple 的差別

Dict 是可變的，Tuple 是不可變的：

  • 可變（Dict）：修改值時，記憶體位址不變，內容原地更新。
  • 不可變（Tuple）：無法修改，若重新賦值則會分配一個全新的記憶體位址。

這樣設計的原因：

  • Dict 為了避免 Hash 碰撞，會預留大量記憶體空間。
  • Tuple 採緊湊排列，佔用空間更小、存取速度更快。
  • 程式碼中看到 Tuple，開發者可以立刻知道「這組資料不會被修改」，有助於協作理解。

使用原則：需要頻繁修改的資料用 Dict，固定的常數或函式回傳值用 Tuple。

可變與不可變型別一覽

不可變型別（Immutable）

型別 說明 範例
int 整數 1, 100, -5
float 浮點數 3.14, 2.0
bool 布林值 True, False
str 字串 "Hello"
tuple 元組 (1, 2, 3)

可變型別（Mutable）

型別 說明 範例
list 列表 [1, 2, 3]
dict 字典 {"a": 1}
set 集合 {1, 2, 3}

如果程式中不斷建立新變數，記憶體中持續新增位址，難道不會爆滿嗎？
