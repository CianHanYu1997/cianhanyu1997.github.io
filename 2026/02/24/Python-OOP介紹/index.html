OOP 是什麼？

全名為物件導向程式設計（Object-Oriented Programming, OOP），是一種程式設計的範式（Paradigm），以物件的方式描述資料應有的狀態與行為。

物件是什麼？

物件通常是指一小塊擁有值的記憶體位置，由**屬性（Attribute）與方法（Method）**組成，通常是由類別（Class）產生的「實體（Instance）」。

物件（Object） = 屬性（Attribute） + 方法（Method）

舉例來說：

概念 說明
Object 小貓
Attribute 白色、體重、生日、公、名字（Honey）
Method 走路、喵喵叫、打翻杯子

在程式設計或執行階段，我們可以藉由更新 Attribute 來改變 Object，以達成預期的結果。例如，當 體重 > 10kg 時，Honey 執行「打翻杯子」的成功率會從 50% 提升至 100%。

類別與物件的關係

類別（Class）就像是一個藍圖或模具，定義了物件應具備哪些屬性與行為。當我們根據這個模具建立實體（Instance）時，雖然它們擁有相同的結構，但每個實體都可以擁有獨立且不同的狀態（資料）。

輸出格式說明：

  • __main__：模組名稱，即當前的執行腳本。
  • .Cat：環境中定義的 Class 名稱。
  • object：一個 Instance，也就是「小貓 Honey」或「小貓 Mochi」。
  • at 0x107f92e80：該 Instance 在記憶體中的實際位址。

__main__ 模組本身是 GC Root，可以回頭複習前篇介紹的垃圾回收機制（GC）。

層級 名稱 角色 說明
Root __main__ 執行環境（Module） 程式運行的起點，永遠不會被回收。
↓ 引用 Cat 類別（Class Object） 實體物件的「模板」。
↓ 引用 honey 變數（Variable） 全域定義的名稱。
↓ 指向 0x107f... 實體（Instance Object） 真正佔用記憶體、會打翻杯子的那隻「貓」。

所有的 Object，不論是自己設計還是 Python 內建的，都是由某個 Class 建立的，可以透過 __class__ 屬性觀察該 Object 是由哪個類別所建立的。

物件的屬性

每個 Object 都有 Attribute 可以存取，且其狀態可以被改變。Class 裡面的 __init__() 負責初始化，通常會將傳入的參數指定為 Object 的 Attribute，讓我們得以區分不同的實體。

在 __init__() 中建立的 Attribute，稱為實例屬性（Instance Attributes）。

物件的實體方法

除了 Attribute 以外，在 Class 裡面定義的 function 稱為方法（Method），或稱實體方法（Instance Method）。

類別屬性

屬於 Class 本身的 Attribute，不需要建立 Instance 即可直接存取。同樣可以透過 __dict__ 查看 Class Attribute。

常見用途：

  1. 資訊共享：讓所有 Instance 共享同一份資料（如 list、dict）。需注意可變型別的共享問題——一個 Instance 修改後，所有 Instance 都會受到影響。
  2. 狀態同步：追蹤所有 Instance 的集體行為。例如爬蟲的頻率限制，所有 Instance 需要共享同一個冷卻時間計數器。
  3. 命名空間封裝：將與這個類別相關的常數或設定集中放置，避免污染全域命名空間。

屬性查找順序：Python 會先在 Instance 中尋找 Attribute，若找不到才往 Class 中查找，最後仍找不到才拋出 AttributeError。這就是為什麼 honey.count 能存取到 Class Attribute count。

屬性裝飾器（Property）

Attribute 雖然方便，但若要防止被賦予不合理的值，可以使用 @property 裝飾器來加以限制。其核心概念是將 Method 偽裝成 Attribute，讓屬性具備資料驗證、動態計算與唯讀保護等功能。

一個完整的 property 通常包含三個組件：

組件 裝飾器 觸發時機
Getter @property 讀取屬性時
Setter @<name>.setter 寫入屬性時
Deleter @<name>.deleter 刪除屬性時

限定屬性存取

__slots__ 可以限制 Object 的 Attribute 只能設定在 __slots__ 的名單裡，如果不在名單中，會跳出 AttributeError 錯誤。

類別方法與靜態方法

除了實體方法之外，Python 還提供兩種特殊的方法型態：靜態方法與類別方法。

方法類型 裝飾器 第一個參數 說明
實體方法 （無） self（實體本身） 操作實體的狀態
靜態方法 @staticmethod 無 與類別相關的工具函數
類別方法 @classmethod cls（類別本身） 操作類別層級的狀態

靜態方法（@staticmethod）

靜態方法不依賴任何實體或類別的狀態，不需要傳入 self 或 cls。雖然它邏輯上隸屬於某個類別，但本質上就是一個放在類別命名空間裡的普通函數。

適用情境：驗證、格式轉換等與物件狀態無關的工具型操作。

類別方法（@classmethod）

類別方法與類別本身綁定，第一個參數固定為 cls（代表類別本身，而非實體）。

常見用途：

1. 工廠方法：提供多種方式來建立 Object Instance。

2. 修改類別狀態：操作所有 Instance 共享的類別屬性。

繼承

常常看到 super()，這是什麼意思？

當我們在子類別定義一個跟父類別一模一樣的方法時，這叫「覆寫 (Override)」。

  • 如果不寫 super()：子類別會完全蓋掉父類別的行為。貓走路就只會印出 "Cat walking"。
  • 寫了 super()：這叫「擴充 (Extend)」，即呼叫父類別的同名方法。

多重繼承

可以同時繼承多個 class，好處是可以組合不同 class 的功能，但缺點是比較難找到某個方法來自哪個 class，不同父類別之間的 method 也可能產生衝突，有衝突時又不確定會先使用誰的 method。

但如果每個 class 都有 __init__，在 SmartPhone 的 __init__ 裡呼叫了 super()，問題來了：會呼叫誰的 __init__？
這時就會啟動 Python 的算法：MRO。
當我們呼叫 super() 時，Python 並不是單純找「父類別」，而是根據一個清單按順序找，這個清單就叫 MRO。
所以 super() 在多重繼承中不是指「父類別」，而是指「MRO 序列中的下一個」。
